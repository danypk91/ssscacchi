1. Installazione ambiente di lavoro

Questa sezione spiega come preparare il tuo PC Windows per lavorare sul sito.

1.1 Installare Git

1
Vai su https://git-scm.com/download/win e scarica l'installer.
2
Esegui l'installer. Lascia tutte le opzioni predefinite e clicca Next fino alla fine.
3
Verifica l'installazione aprendo il Terminale (cerca "cmd" o "PowerShell" nel menu Start): 
git --version
Deve mostrare qualcosa come git version 2.47.0.

1.2 Installare Node.js

1
Vai su https://nodejs.org e scarica la versione LTS (quella consigliata).
2
Esegui l'installer con le opzioni predefinite.
3
Verifica: 
node --version
npm --version
Node deve essere almeno v22.

1.3 Installare un editor di testo

Consigliamo Visual Studio Code (gratuito):

  1. Scarica da https://code.visualstudio.com
  2. Installa e aprilo
  3. Installa l'estensione Astro (cerca "Astro" nel pannello estensioni)

1.4 Scaricare il progetto

Apri il terminale e scegli dove salvare il progetto (es. Desktop):

# Spostati sul Desktop
cd %USERPROFILE%\Desktop

# Clona il progetto da GitHub
git clone https://github.com/danypk91/ssscacchi.git

# Entra nella cartella
cd ssscacchi

# Installa le dipendenze
npm install
Nota: Per clonare il progetto serve accesso al repository GitHub. Se non hai un account GitHub, creane uno su github.com e chiedi di essere aggiunto come collaboratore.

2. Avviare il sito in locale

Dopo aver scaricato il progetto, puoi vedere il sito sul tuo computer.

Modalita sviluppo (consigliata per lavorare)

npm run dev

Apri il browser su http://localhost:4321. Il sito si aggiorna automaticamente quando salvi un file.

Creare la versione di produzione

npm run build

Anteprima della versione di produzione

npm run preview
Per fermare il server: premi Ctrl + C nel terminale.

4. Scrivere in Markdown

I contenuti del sito (notizie e tornei) sono scritti in Markdown, un linguaggio semplice per formattare il testo. Ecco una guida completa.

4.1 Titoli

# Titolo principale (H1)
## Sottotitolo (H2)
### Sotto-sottotitolo (H3)
#### Titolo piccolo (H4)
Regola: nei file delle notizie/tornei, usa ## come livello massimo (il titolo H1 viene dal frontmatter).

4.2 Testo formattato

Testo normale.

Testo in **grassetto**.

Testo in *corsivo*.

Testo in ***grassetto e corsivo***.

Testo ~~barrato~~.

Risultato: Testo normale. Testo in grassetto. Testo in corsivo.

4.3 Paragrafi e a capo

Questo e' il primo paragrafo.

Questo e' il secondo paragrafo.
(Serve una riga vuota per separare i paragrafi)

Per andare a capo senza nuovo paragrafo,
metti due spazi alla fine della riga.

4.4 Elenchi puntati

- Primo elemento
- Secondo elemento
- Terzo elemento
  - Sotto-elemento (indentato con 2 spazi)
  - Altro sotto-elemento

4.5 Elenchi numerati

1. Primo passo
2. Secondo passo
3. Terzo passo

4.6 Link

Link a un sito esterno:
[Testo del link](https://www.esempio.it)

Esempio reale:
Iscrizioni su [VeSuS](https://vesus.org/tournament/xxx).

Risultato: Iscrizioni su VeSuS.

4.7 Testo in evidenza (citazione)

> Questo testo appare come citazione evidenziata.
> Puo' continuare su piu' righe.

4.8 Linea separatrice

Testo sopra

---

Testo sotto

5. Aggiungere una notizia

Passo 1: Crea il file

Crea un nuovo file nella cartella src/content/notizie/. Il nome del file diventa l'indirizzo web (URL), quindi usa lettere minuscole, trattini e niente spazi:

src/content/notizie/risultati-sigona-2026.md

Passo 2: Scrivi il frontmatter

All'inizio del file, tra i due ---, inserisci i metadati obbligatori:

---
title: "Risultati Memorial Sigona 2026"
date: "2026-05-18"
excerpt: "Grande successo per la XX edizione con 35 giovani scacchisti."
---
CampoObbligatorioDescrizione
titleSiTitolo della notizia
dateSiData in formato AAAA-MM-GG
excerptSiRiassunto breve (appare nell'elenco notizie)

Passo 3: Scrivi il contenuto

Dopo il secondo ---, scrivi il testo in Markdown:

---
title: "Risultati Memorial Sigona 2026"
date: "2026-05-18"
excerpt: "Grande successo per la XX edizione con 35 giovani scacchisti."
---

## Cronaca del torneo

Sabato 18 maggio si e' svolta la **XX edizione del Memorial Sigona**
presso la sede del circolo, con la partecipazione di 35 giovani
scacchisti provenienti da tutta la Liguria.

6. Aggiungere un torneo

Passo 1: Crea il file

src/content/tornei/sociale-autunno-2026.md

Passo 2: Scrivi il frontmatter

---
title: "Torneo Sociale di Autunno 2026"
year: 2026
date: "2026-09-15"
excerpt: "7 turni su 7 giornate"
bando: "tornei/2026/sociale-autunno-2026_bando.pdf"
vesus: "https://vesus.org/tournament/abc123"
---
CampoObbligatorioDescrizione
titleSiNome completo del torneo
yearSiAnno (numero, senza virgolette)
dateSiData inizio in formato AAAA-MM-GG
excerptSiFormato breve (es. "7 turni su 7 giornate")
bandoNoPercorso del PDF su R2 (vedi sotto)
classificaNoURL della classifica
vesusNoLink alla pagina VeSuS del torneo
fotoAlbumNoPercorso album foto (es. "/fotografie/2026/album")

Passo 3: Scrivi il contenuto

---
title: "Torneo Sociale di Autunno 2026"
year: 2026
date: "2026-09-15"
excerpt: "7 turni su 7 giornate"
bando: "tornei/2026/sociale-autunno-2026_bando.pdf"
vesus: "https://vesus.org/tournament/abc123"
---

## Dettagli

- **Formato:** 7 turni su 7 giornate
- **Cadenza:** 60 minuti + 30 secondi per mossa
- **Orario:** ogni martedi alle 21:15
- **Sede:** Via Donghi 8, Genova

## Calendario

1. **1o turno** — 15 settembre
2. **2o turno** — 22 settembre
3. **3o turno** — 29 settembre
4. **4o turno** — 6 ottobre
5. **5o turno** — 13 ottobre
6. **6o turno** — 20 ottobre
7. **7o turno** — 27 ottobre

Caricare il PDF del bando su R2

Se il torneo ha un bando PDF, caricalo su Cloudflare R2 prima di pubblicare:

# Da terminale, nella cartella del progetto:
npx wrangler r2 object put "santasabina/tornei/2026/sociale-autunno-2026_bando.pdf" \
  --file="percorso/del/bando.pdf" \
  --content-type="application/pdf" \
  --remote
Importante: il valore del campo bando nel frontmatter deve corrispondere al percorso su R2 senza il nome del bucket. Esempio: se carichi su santasabina/tornei/2026/bando.pdf, nel frontmatter scrivi tornei/2026/bando.pdf.

7. Aggiungere foto

Le foto sono organizzate su Cloudflare R2 con questa struttura:

foto/
  2026/
    2026_sociale_primavera/
      foto1.jpg
      foto2.jpg
      ...
    2026_memorial_bertolini/
      foto1.jpg
      ...

Caricare un album fotografico

Supponiamo di avere una cartella di foto del Torneo Sociale Primavera 2026:

# Carica ogni foto singolarmente:
npx wrangler r2 object put "santasabina/foto/2026/2026_sociale_primavera/foto1.jpg" \
  --file="foto1.jpg" \
  --content-type="image/jpeg" \
  --remote

npx wrangler r2 object put "santasabina/foto/2026/2026_sociale_primavera/foto2.jpg" \
  --file="foto2.jpg" \
  --content-type="image/jpeg" \
  --remote

# Oppure, per caricare tutte le foto di una cartella in un colpo solo (PowerShell):
Get-ChildItem .\mie_foto\*.jpg | ForEach-Object {
  npx wrangler r2 object put "santasabina/foto/2026/2026_sociale_primavera/$($_.Name)" `
    --file="$($_.FullName)" `
    --content-type="image/jpeg" `
    --remote
}

Convenzioni per i nomi

  • Nome cartella album: {anno}_{nome_evento} (es. 2026_sociale_primavera)
  • Usa lettere minuscole, numeri e underscore _
  • No spazi, no caratteri speciali, no accenti
Le pagine galleria si generano automaticamente! Non serve creare nessun file. Basta caricare le foto su R2 e fare un nuovo deploy (vedi sezione 8). Il sito crea da solo la pagina dell'anno e dell'album.

Collegare un album a un torneo

Per far apparire il pulsante "Fotografie" nella pagina del torneo, aggiungi il campo fotoAlbum nel frontmatter:

---
title: "Torneo Sociale di Primavera 2026"
year: 2026
date: "2026-05-01"
excerpt: "7 turni su 7 giornate"
fotoAlbum: "/fotografie/2026/2026_sociale_primavera"
---

8. Pubblicare le modifiche

Dopo aver modificato i file, devi inviarli su GitHub. Vercel pubblichera il sito automaticamente.

Metodo 1: Da terminale

# 1. Controlla cosa hai modificato
git status

# 2. Aggiungi i file modificati
git add src/content/notizie/risultati-sigona-2026.md
git add src/content/tornei/sociale-autunno-2026.md

# 3. Crea il commit (descrivi brevemente cosa hai fatto)
git commit -m "Aggiunti risultati Sigona e torneo autunno 2026"

# 4. Invia a GitHub
git push

Dopo il push, attendi 1-2 minuti e il sito sara aggiornato su santasabina.vercel.app.

Metodo 2: Da Visual Studio Code

  1. Apri il pannello Source Control (icona ramificazione, terza nella barra laterale sinistra)
  2. Vedrai i file modificati. Clicca il + accanto a ciascuno per aggiungerli
  3. Scrivi un messaggio di commit nel campo di testo in alto
  4. Clicca il pulsante Commit
  5. Clicca Sync Changes per inviare a GitHub

Forzare il deploy senza modifiche al codice

Se hai solo caricato foto o PDF su R2 (senza modificare file .md), devi forzare un redeploy:

  1. Vai su vercel.com → progetto ssscacchi
  2. Vai nella tab Deployments
  3. Clicca i tre puntini (...) sull'ultimo deploy
  4. Seleziona Redeploy
